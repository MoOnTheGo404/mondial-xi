# Methodology

## Temporal integrity (the core discipline)

All state (Elo, form, rolling goals, rest, congestion) lives in a single
forward pass over matches sorted by date. For each match the feature vector is
**emitted before** the engine updates with that match's result. Consequences,
each locked in by `tests/unit/ml/test_leakage.py`:

- altering a *future* result cannot change an earlier feature vector;
- altering a match's *own* result cannot change its features;
- truncating the dataset after match *k* leaves match *k*'s features intact.

There are no ranking releases, market odds, or post-match statistics in the
feature set, so no auxiliary leakage channels exist. Historical availability
data does not exist in the open data — see docs/availability-model.md for how
we avoid manufacturing it.

## Ratings

Elo with: tournament-importance K (World Cup 60 → friendly 20, scaled by a
tuned global factor), margin-of-victory multiplier (1 / 1.5 / (11+N)/8),
tuned home advantage. Grid search over home_advantage ∈ {60, 80, 100} ×
k_scale ∈ {0.75, 1.0, 1.25} selected by **validation log loss of the
Elo-only logistic model** (result: 80 / 0.75 — see
`ml/artifacts/elo_tuning.json`). New teams start at 1500; pre-1980 matches
warm state but never enter model fitting.

## Features

`elo_diff_eff` (incl. venue adjustment), raw Elos, exponentially-weighted
form (half-life 4 matches), rolling goals for/against (last 10), rest days
(capped 60), matches in last 365 days, match counts (experience), neutral
flag, competition importance. Weather/altitude/travel were evaluated against
the bar "reliably available historically AND prospectively" and excluded
(no feasible historical backfill within Open-Meteo fair use) — they appear
as display/scenario context only.

## Models & selection

frequency baseline → Elo logistic → independent Poisson GLMs → Dixon–Coles
(ρ by MLE) → gradient-boosted trees (+ per-class isotonic calibration fitted
only on validation). Chronological protocol: fit 1980–2018, select on
2019–2022, report once on 2023→cutoff. The champion (calibrated GBM by
validation log loss) was frozen before the test window was evaluated. The
honest outcome — Dixon–Coles marginally better on test — is reported as-is;
top models sit within ~0.004 log loss because international outcomes are
dominated by rating differentials.

## Scenario adjustments

Player unavailability subtracts the player's shrunken attack impact from that
side's expected goals; DC matrices are recomputed and the champion's
probabilities are tilted by `P_champion × (Q_adj / Q_base)` renormalized —
transparent, monotonic, and reported next to the untouched team-only
forecast. Doubtful players contribute `(1 − p_available) × impact`
(marginalization, not a hard assumption).

## Knockout math

P(extra time) = champion draw probability; ET scorelines use DC with means
scaled ⅓; shootouts 50/50 (no defensible skill signal in the data —
documented). Advancement = P(win) + P(draw)·(P(win ET) + P(ET draw)·0.5).

## Evaluation metrics

Multiclass log loss, ranked probability score, multiclass Brier, top-pick
accuracy, ECE + per-class reliability bins, confidence buckets, slices by
year / competition tier / confederation / favorites-vs-close. All artifacts
generated by `kickoff_ml.evaluation.run`; the README and website load them
programmatically.

## Track record

Immutable snapshots (SHA-256 content hash) for genuinely-upcoming fixtures
only; scoring appends result columns without touching payloads; backtests are
always labeled backtests. There is no code path that creates a "published"
forecast retroactively.
